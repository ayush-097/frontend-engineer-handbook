# Glossary

Quick reference for frontend terminology.

## A
**Accessibility (a11y)** - Ensuring web apps are usable by people with disabilities.

**ARIA** - Accessible Rich Internet Applications. Attributes to improve accessibility.

## B
**Bundler** - Tool that combines modules into optimized files (webpack, Vite).

## C
**CLS (Cumulative Layout Shift)** - Metric measuring visual stability.

**CSP (Content Security Policy)** - Security header preventing XSS attacks.

**CSRF (Cross-Site Request Forgery)** - Attack exploiting user authentication.

## D
**DOM** - Document Object Model. Browser's representation of HTML.

## E
**Event Loop** - JavaScript's mechanism for handling async operations.

## F
**FCP (First Contentful Paint)** - When first content appears on screen.

**FID (First Input Delay)** - Time from first interaction to browser response.

## H
**Hydration** - Attaching event listeners to server-rendered HTML.

## L
**LCP (Largest Contentful Paint)** - When main content is rendered.

**Lazy Loading** - Deferring resource loading until needed.

## M
**Memoization** - Caching function results for performance.

**Micro-frontend** - Architectural pattern dividing frontend into independent apps.

## R
**Reconciliation** - React's process of comparing virtual DOMs.

**RUM (Real User Monitoring)** - Collecting performance data from real users.

## S
**SSR (Server-Side Rendering)** - Rendering HTML on the server.

**SSG (Static Site Generation)** - Pre-rendering pages at build time.

## T
**Tree Shaking** - Removing unused code from bundles.

**TTI (Time to Interactive)** - When page becomes fully interactive.

## V
**Virtual DOM** - In-memory representation of DOM for efficient updates.

**Virtualization** - Rendering only visible items in long lists.

## W
**Web Vitals** - Google's metrics for user experience (LCP, FID, CLS).

**XSS (Cross-Site Scripting)** - Injecting malicious scripts into web pages.
