## ðŸ’¡ General Tips

### Coding Interviews
1. **Listen carefully** to the problem
2. **Ask questions** before coding
3. **Start with brute force**, optimize later
4. **Think out loud** constantly
5. **Test with examples** before submitting

### System Design
1. **Clarify requirements** (functional & non-functional)
2. **Start high-level**, drill down
3. **Discuss tradeoffs** explicitly
4. **Draw diagrams** as you talk
5. **Scale iteratively** (start simple)

### Behavioral
1. **Use STAR method** (Situation, Task, Action, Result)
2. **Be specific**, not general
3. **Show impact** with numbers
4. **Be honest** about failures
5. **Tie to role** you're interviewing for
